# Настройка ИИ-бота для сайта

## Бесплатные варианты ИИ API

### Вариант 1: Google Gemini API (Рекомендуется) ⭐

**Преимущества:**
- Полностью бесплатный tier с хорошими лимитами
- Качественные ответы
- Простая интеграция
- Поддержка русского, английского и казахского языков

**Шаги настройки:**

1. Перейдите на https://makersuite.google.com/app/apikey
2. Войдите в Google аккаунт
3. Нажмите "Create API Key"
4. Скопируйте ключ
5. Создайте файл `.env.local` в корне проекта:
   ```
   GEMINI_API_KEY=ваш_ключ_здесь
   ```
6. Перезапустите сервер разработки

### Вариант 2: Hugging Face Inference API

**Преимущества:**
- Полностью бесплатный
- Много моделей на выбор

**Шаги настройки:**

1. Перейдите на https://huggingface.co/settings/tokens
2. Создайте новый токен (Read access)
3. Добавьте в `.env.local`:
   ```
   HUGGINGFACE_API_KEY=ваш_токен_здесь
   ```
4. Перезапустите сервер

### Вариант 3: OpenAI API (если есть бесплатные кредиты)

Если у вас есть бесплатные кредиты OpenAI:

1. Получите ключ на https://platform.openai.com/api-keys
2. Добавьте в `.env.local`:
   ```
   OPENAI_API_KEY=ваш_ключ_здесь
   ```
3. Обновите `app/api/chat/route.ts` для поддержки OpenAI

## Без API ключа

Если не настроить API ключ, бот будет работать в режиме fallback с простой логикой на основе ключевых слов. Это не полноценный ИИ, но базовые вопросы о ценах будут работать.

## Обновление цен и услуг

Все цены и услуги хранятся в:
- `app/data/pricing.json` - структурированные данные о ценах
- `app/api/chat/route.ts` - системный промпт с информацией о ценах

Чтобы обновить цены:
1. Отредактируйте `app/data/pricing.json`
2. Обновите `SYSTEM_PROMPT` в `app/api/chat/route.ts`
3. Перезапустите сервер

## Настройка переменных окружения на Vercel 🚀

⚠️ **Важно**: Файл `.env.local` работает только локально! На Vercel нужно настроить переменные окружения отдельно.

### Почему `.env.local` не работает на Vercel?

1. Файл `.env.local` находится в `.gitignore` и не попадает в репозиторий
2. Vercel не загружает `.env.local` файлы автоматически
3. Переменные окружения нужно настроить в панели управления Vercel

### Как настроить переменные окружения на Vercel:

#### Способ 1: Через веб-интерфейс Vercel (Рекомендуется)

1. Откройте ваш проект на [vercel.com](https://vercel.com)
2. Перейдите в **Settings** (Настройки)
3. Выберите **Environment Variables** (Переменные окружения)
4. Добавьте переменные:
   - **Name**: `GEMINI_API_KEY`
   - **Value**: ваш API ключ
   - **Environment**: выберите `Production`, `Preview`, и/или `Development` (рекомендуется выбрать все)
5. Нажмите **Save**
6. **Важно**: После добавления переменных нужно передеплоить проект:
   - Перейдите в **Deployments**
   - Найдите последний деплой
   - Нажмите на три точки (⋯) → **Redeploy**

#### Способ 2: Через Vercel CLI

```bash
# Установите Vercel CLI (если еще не установлен)
npm i -g vercel

# Добавьте переменную окружения
vercel env add GEMINI_API_KEY

# Введите значение при запросе
# Выберите окружения (Production, Preview, Development)

# Передеплойте проект
vercel --prod
```

#### Способ 3: Через файл `.env` (не рекомендуется для продакшена)

Можно создать файл `.env` (без `.local`), но он будет в Git, что небезопасно для API ключей.

### Проверка переменных окружения на Vercel

После деплоя проверьте логи:
1. Откройте **Deployments** → выберите последний деплой
2. Откройте **Functions** → выберите `api/chat`
3. Проверьте логи - должны быть сообщения:
   ```
   ✅ GEMINI_API_KEY loaded: true
   ✅ Используется Gemini API
   ```

Если видите `⚠️ API ключ не найден`, значит переменная не настроена правильно.

### Переменные для разных окружений

На Vercel можно настроить разные значения для:
- **Production** - основной сайт
- **Preview** - превью для pull requests
- **Development** - локальная разработка (обычно не используется)

Рекомендуется использовать один и тот же ключ для всех окружений.

## Безопасность

⚠️ **Важно**: 
- Никогда не коммитьте `.env.local` в Git! Он уже добавлен в `.gitignore`.
- Не добавляйте API ключи в код или в публичные файлы
- Используйте переменные окружения Vercel для продакшена

## Поддержка

Если возникли проблемы:
1. Проверьте консоль браузера (F12)
2. Проверьте логи сервера на Vercel
3. Убедитесь, что API ключ правильный
4. Проверьте, что файл `.env.local` создан в корне проекта (для локальной разработки)
5. **Для Vercel**: Проверьте, что переменные окружения добавлены в настройках проекта и проект передеплоен

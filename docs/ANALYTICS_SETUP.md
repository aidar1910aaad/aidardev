## 🔧 Что нужно настроить:

### 1. Google Analytics 4 (GA4) - КРИТИЧНО

**Зачем:**
- Детальная аналитика поведения пользователей
- Конверсии и цели
- Интеграция с Google Search Console
- Бесплатно и мощно

**Как настроить:**

1. **Создайте аккаунт GA4:**
   - Перейдите на https://analytics.google.com
   - Создайте аккаунт и свойство
   - Получите Measurement ID (формат: G-XXXXXXXXXX)

2. **Добавьте в переменные окружения:**
   ```env
   NEXT_PUBLIC_GA_ID=G-XXXXXXXXXX
   ```

3. **Компонент уже добавлен в layout.tsx** ✅
   - Автоматически подключится после добавления переменной

**Что будет отслеживаться:**
- Посетители, сессии, просмотры страниц
- События (клики, скроллы, формы)
- Конверсии
- Демография и интересы
- Источники трафика

---

### 2. Яндекс.Метрика - ВАЖНО для KZ/RU рынка

**Зачем:**
- Лучшая аналитика для российского рынка
- Heatmaps и вебвизор (запись сессий)
- Детальная аналитика поведения
- Бесплатно

**Как настроить:**

1. **Создайте счетчик:**
   - Перейдите на https://metrika.yandex.ru
   - Создайте новый счетчик
   - Получите ID счетчика (число, например: 12345678)

2. **Добавьте в переменные окружения:**
   ```env
   NEXT_PUBLIC_YANDEX_METRIKA_ID=12345678
   ```

3. **Компонент уже добавлен в layout.tsx** ✅
   - Автоматически подключится после добавления переменной

**Что будет отслеживаться:**
- Посетители и просмотры
- Heatmaps (карты кликов)
- Вебвизор (запись сессий пользователей)
- Детальная аналитика поведения
- A/B тестирование

---

### 3. Microsoft Clarity - ОПЦИОНАЛЬНО (но полезно)

**Зачем:**
- Бесплатные heatmaps и session recordings
- Анализ поведения пользователей
- Выявление проблем UX
- Полностью бесплатно

**Как настроить:**

1. **Создайте проект:**
   - Перейдите на https://clarity.microsoft.com
   - Создайте новый проект
   - Получите Project ID (строка, например: abc123def456)

2. **Добавьте в переменные окружения:**
   ```env
   NEXT_PUBLIC_CLARITY_ID=abc123def456
   ```

3. **Компонент уже добавлен в layout.tsx** ✅
   - Автоматически подключится после добавления переменной

**Что будет отслеживаться:**
- Heatmaps (клики, скроллы, движения мыши)
- Session recordings (запись сессий)
- Анализ проблем UX
- Dead clicks и rage clicks

## 📝 Настройка переменных окружения

Создайте файл `.env.local` в корне проекта:

```env
# Сайт
NEXT_PUBLIC_SITE_URL=https://aidar.dev

# Google
NEXT_PUBLIC_GA_ID=G-XXXXXXXXXX
NEXT_PUBLIC_GOOGLE_VERIFICATION=your-verification-code

# Яндекс
NEXT_PUBLIC_YANDEX_METRIKA_ID=12345678
NEXT_PUBLIC_YANDEX_VERIFICATION=your-verification-code

# Microsoft Clarity (опционально)
NEXT_PUBLIC_CLARITY_ID=abc123def456
```

**Важно:** 
- Не коммитьте `.env.local` в git (уже в .gitignore)
- Добавьте переменные в Vercel Dashboard → Settings → Environment Variables

## 🔍 Регистрация в поисковых системах

### Google Search Console

1. **Перейдите на:** https://search.google.com/search-console
2. **Добавьте свойство:**
   - Введите URL сайта: `https://aidar.dev`
   - Выберите метод проверки (HTML тег или файл)
3. **Используйте verification код:**
   - Добавьте в `.env.local`: `NEXT_PUBLIC_GOOGLE_VERIFICATION=your-code`
   - Или загрузите HTML файл
4. **Отправьте sitemap:**
   - URL: `https://aidar.dev/sitemap.xml`

### Яндекс.Вебмастер

1. **Перейдите на:** https://webmaster.yandex.ru
2. **Добавьте сайт:**
   - Введите URL: `https://aidar.dev`
   - Выберите метод проверки
3. **Используйте verification код:**
   - Добавьте в `.env.local`: `NEXT_PUBLIC_YANDEX_VERIFICATION=your-code`
4. **Отправьте sitemap:**
   - URL: `https://aidar.dev/sitemap.xml`

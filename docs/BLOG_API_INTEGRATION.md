# Интеграция блога с внешним API

## Выбранный подход: ISR (Incremental Static Regeneration)

**Почему ISR:**
- ✅ Статьи быстро загружаются (предсгенерированный HTML)
- ✅ Автоматически обновляются (раз в час или при деплое)
- ✅ Полностью индексируются Google (готовый HTML)
- ✅ Новые статьи появляются автоматически без пересборки всего сайта

---

### 2. Endpoint для получения одной статьи

**URL:** `GET /api/blog/posts/:slug`  
**Пример:** `GET /api/blog/posts/kak-uvelichit-konversiyu-sajta-2024`

**Ответ (успех):**
```json
{
  "post": {
    "id": "1",
    "slug": "kak-uvelichit-konversiyu-sajta-2024",
    "title": { "ru": "...", "en": "...", "kz": "..." },
    "description": { "ru": "...", "en": "...", "kz": "..." },
    "excerpt": { "ru": "...", "en": "...", "kz": "..." },
    "category": { "ru": "...", "en": "...", "kz": "..." },
    "date": "2024-01-15",
    "keywords": { "ru": [...], "en": [...], "kz": [...] },
    "readingTime": 8,
    "published": true,
    "content": { "ru": "...", "en": "...", "kz": "..." }
  }
}
```

**Ответ (не найдено):**
```json
{
  "error": "Post not found"
}
```

### 3. Важные требования

1. **CORS** - должен разрешать запросы с вашего домена
2. **Published filter** - возвращать только статьи где `published: true`
3. **Сортировка** - статьи должны быть отсортированы по дате (новые первые)
4. **Кодировка** - UTF-8 для корректного отображения русского и казахского текста
5. **Скорость** - ответ должен быть быстрым (желательно < 500ms)

## Как это будет работать

1. **При сборке сайта (`npm run build`):**
   - Next.js запрашивает все статьи через `GET /api/blog/posts`
   - Генерирует HTML для каждой статьи
   - Сохраняет статические файлы

2. **При деплое:**
   - Статический HTML загружается быстро
   - Пользователи видят готовые страницы

3. **Автообновление (ISR):**
   - Раз в час Next.js проверяет новые статьи
   - Если есть новые - генерирует для них HTML
   - Без пересборки всего сайта

4. **Google индексация:**
   - Робот получает готовый HTML
   - Видит весь контент сразу
   - Индексирует как обычные статические страницы
